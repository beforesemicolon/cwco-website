LiveCycles

The native web component API has livecycles methods:

  • connectedCallback;
  • disconnectedCallback;
  • attributeChangedCallback;
  • adoptedCallback;

WebComponent simply renames them and "fixes" couple of things about some of them.

onMount

This livecycle function is called when the component is inserted in the document soon after the first render.

class BFSButton extends WebComponent { onMount() { console.log('mounted', this.mounted, this.isConnected) } } BFSButton.register(); // will trigger onMount call document.body.appendChild(new BFSButton())

You can verify if the component is in the DOM by checking the native isConnected or WebComponend "mounted" properties. We would strongly you to use mounted property instead of isConnected as it will better tell you if the component is in the DOM and its template was parsed and set as well.
